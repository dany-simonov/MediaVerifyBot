# Источник

**Платформа верификации медиаконтента на основе ИИ**

Источник определяет, создан ли контент человеком или сгенерирован искусственным интеллектом. Поддерживаются фото, аудио, видео и текст. Результат включает вердикт, индекс подлинности и объяснение.

🌐 **Сайт:** [istochnik.appwrite.network](https://istochnik.appwrite.network)  
📧 **Связаться:** [istochnik-media@yandex.com](mailto:istochnik-media@yandex.com)

---

## Статус и стратегия

Мы отказались от Telegram-канала как основного входа для пользователей и сфокусировались на веб-платформе. Проект мигрирован на Appwrite Cloud: клиент, хранение и функция анализа работают через Appwrite SDK и Appwrite Functions.

---

## Что реализовано сейчас

- **Веб-кабинет** с авторизацией, историей проверок и отдельной страницей для глубокой проверки текста
- **Большая проверка текста**: детектор ИИ + фактчек/заимствования с пословной подсветкой
- **Big Check**: пакетный кросс-анализ медиафайлов
- **Explainable AI**: осмысленные объяснения с источниками

---

## Реальные модели и цепочки анализа

### Фото
- **Sightengine (genai)** — основная модель
- **HuggingFace** (fallback): `dima806/deepfake-vs-real-image-detection`

### Аудио
- **Resemble Detect** — основная модель
- **HuggingFace** (fallback): `mo-gg/wav2vec2-large-xlsr-deepfake-detection`

### Видео
- **FFmpeg + Sightengine** — анализ
- **HuggingFace Image** fallback при ошибках/лимитах

### Текст
- **Sapling AI** — детекция AI-текста
- **g4f hybrid fact-check** — фактчек/заимствования
	- Каскад моделей: `gpt-5-nano` → `gpt-oss-120b` → `command-r`

Если основной провайдер недоступен, используется fallback, чтобы сохранить стабильность.

# Istochnik

**AI-powered media verification platform**

Istochnik determines whether content was created by a human or generated by AI. It supports images, audio, video, and text. The result includes a verdict, an authenticity index, and an explanation.

🌐 **Website:** [istochnik.appwrite.network](https://istochnik.appwrite.network)  
📧 **Contact:** [istochnik-media@yandex.com](mailto:istochnik-media@yandex.com)

---

## Project status

- We dropped Telegram as the primary entry point and focused on the web platform.
- The project was migrated to Appwrite Cloud: auth, storage, and analysis are handled via Appwrite SDK and Functions.
- The legacy PostgreSQL layer was removed after migration to Appwrite.
- The public API is in development.

---

## What is implemented

- Web dashboard with authentication and check history
- Deep Text Check (AI detection + fact-check/plagiarism with word-level highlighting)
- Big Check: batch cross-analysis for multiple media files
- Explainable AI: result explanations with sources
- Frame-by-frame video analysis with aggregation

---

## Real models and analysis chains

### Images
- Sightengine (genai) — primary model
- HuggingFace fallback: `dima806/deepfake-vs-real-image-detection`

### Audio
- Resemble Detect — primary model
- HuggingFace fallback: `mo-gg/wav2vec2-large-xlsr-deepfake-detection`

### Video
- FFmpeg + Sightengine — frame-by-frame analysis
- HuggingFace Image fallback on errors/limits

### Text
- Sapling AI — AI-text detection
- g4f hybrid fact-check — fact-check/plagiarism
	- Model cascade: `gpt-4.1-nano` → `gpt-oss-120b` → `command-r`

If the primary provider is unavailable, a fallback is used to keep the pipeline stable.

---

## Architecture and Appwrite

- Frontend: React 18, TypeScript, Vite, Tailwind CSS, Zustand, React Router
- Appwrite SDK: auth, storage, function execution
- Appwrite Functions: single analysis entrypoint (`src/main.py`), Python 3.14 runtime
- Local FastAPI is used for internal tasks and testing

The Appwrite migration unified deployment, storage, and auth without a custom production server layer.

---

## Limitations and quality

- The analysis is probabilistic and can be wrong — use results as guidance
- Text accuracy improves with longer inputs (recommended 200–2000 characters)
- Video analysis is limited by size and duration in the pipeline
